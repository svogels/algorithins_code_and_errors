Meet the Game: Pong!

We will use the game Pong to learn about code. Pong is a simple game where you move a paddle to hit a ball.

Code tells the computer how the ball moves and how the paddles work.

What are Code Errors?

An error (or a "bug") is a mistake in the code. It can stop the program from working or make it do the wrong thing.

We will learn about three types of errors.

1. Syntax Errors

A syntax error is like a spelling or grammar mistake in a sentence. The computer does not understand the instruction because it is not written correctly.

Example:

Look at this code. It has a syntax error.

1  # This function should move the ball
2  def move_ball()
3      ball_x = ball_x + 5

What's the mistake?

Line 2 is missing a colon (:) at the end. The computer needs the colon to understand it's a function.

Correct code: def move_ball():

2. Logic Errors

A logic error is when the code runs, but it does the wrong thing. The computer follows the instructions, but the instructions are wrong.

Example:

In Pong, the ball should bounce off the paddle. This code has a logic error.

1  # When the ball hits the paddle
2  if ball_hits_paddle:
3      ball_speed = ball_speed + 5

What's the mistake?

Line 3 makes the ball get faster every time it hits the paddle. It should reverse direction, not speed up.

Correct code: ball_speed = -ball_speed (This makes the speed negative, so it goes the other way).

3. Runtime Errors

A runtime error happens when the program is running and it is asked to do something impossible. This makes the program crash.

Example:

This code tries to get a score from a list. It has a runtime error.

1  # There are two players, Player 1 and Player 2
2  scores = [10, 15]  # Player 1 has 10, Player 2 has 15
3  
4  # This line tries to get the score for Player 3
5  player_3_score = scores[2]

What's the mistake?

The list scores only has two items (at position 0 and 1). There is no item at position 2. Asking for it is impossible and crashes the program.
